# Marp スライドクリエイター

メモ・ドラフト・アウトラインをもとに Marp スライドを作る。
最大の特徴は「まず発表の骨格を固めてからモノを作る」こと。これを守ることで、後から構成が崩れるリスクを防ぐ。

## フロー概要

1. メモを読んで不足情報を確認する
2. **Takeaway をじっくり固める**（ここが最重要）
3. スライドタイトルのみでアウトラインを設計し、ユーザーの承認を得る
4. `task new` でテンプレートを作成してスライドを書く
5. `task compile` でHTML/PDFを出力する

---

## ステップ1: 情報収集と確認

メモを受け取ったら、以下の2点が明記されているかチェックする。どちらかでも不明なら**必ずユーザーに確認してから先に進む**：

- **発表時間**: 何分の発表か？（枚数の目安: 1分あたり1〜2枚）
- **Takeaway**: 発表を聞いた後、聴衆に何を持ち帰ってほしいか？

---

## ステップ2: Takeaway をグリルする

Takeaway は発表の骨子。ここがぼんやりしていると、どんなにスライドを綺麗に作っても「結局何が言いたかったの？」という資料になる。このステップに時間をかける価値がある。

以下の問いを使って深掘りし、Takeaway を**1〜2文の鋭い主張**に絞り込む。全部聞く必要はなく、ユーザーの答えを見ながら必要なものを選ぶ：

- 「その Takeaway を聞いた聴衆は、具体的にどんな行動・判断をすると思いますか？」
- 「今日の発表で一番伝えたい相手は誰ですか？その人は今どんな状況・認識にいますか？」
- 「似たような発表と比べて、この発表だけが言えることは何ですか？」
- 「発表の翌日、聴衆が同僚に話すとしたら、何を一言で伝えると思いますか？」
- 「その結論を聴衆が信じるための根拠は何ですか？その根拠はスライドの中にありますか？」

**Takeaway が固まった判断基準**：
- 1〜2文で言い切れる
- 「〜だから〜できる／すべき／わかった」という因果や示唆が含まれる
- 発表全体がその結論に向かって一本の流れで繋がれる

固まったと思ったら復唱してユーザーに確認する。「これで合ってますか？」と必ず一言聞く。

---

## ステップ3: スライドアウトライン設計

Takeaway と発表時間が確定したら、スライドタイトルのリストを設計する。

**設計の原則**：
- タイトル一覧だけで発表内容の80%が伝わること（タイトルが体言止めの「箇条書き」にならないよう、内容を示唆する言葉を選ぶ）
- 流れは「問題提起 → 原因・背景 → 解決策・提案 → 根拠・事例 → 結論」が基本。メモの内容に合わせて変形する
- セクション見出しは3〜4個が目安
- 総枚数の目安: 発表時間（分） × 1〜2 枚（アジェンダ・タイトル・締め含む）

**アウトラインの出力フォーマット**（このまま出力する）：

```
【発表タイトル】
【発表時間】X分
【Takeaway】〜〜

---スライド構成---
[タイトルスライド]
[アジェンダ]
■ セクション1: xxxxxx
  - スライドタイトル
  - スライドタイトル
■ セクション2: xxxxxx
  - スライドタイトル
  - スライドタイトル
■ セクション3: xxxxxx
  - スライドタイトル
  - スライドタイトル
[まとめ]
[締めスライド]
```

アウトラインを出したら「このアウトラインでGoですか？修正があれば教えてください」と確認する。

**Goが出るまで、スライドの中身（本文・箇条書き・デザイン）には一切手をつけない。**

---

## ステップ4: スライド作成

Goが出たら以下の手順で進める。

### 4-1. テンプレート作成

```bash
task new -- <発表タイトル>
```

作成されたディレクトリ名（`slides/YYYYMMDD_title`）を控えておく。

### 4-2. スライド執筆

`slides/YYYYMMDD_title/main.md` を編集してスライドを書く。

スライドの構成ルールはプロジェクトの CLAUDE.md に従う。よく使う要素：

| やりたいこと | 使うクラス・要素 |
|---|---|
| タイトルスライド・締め | `<!-- _class: title -->` |
| セクション区切り | `<!-- _class: section -->` |
| 2カラムレイアウト | `<!-- _class: cols -->` + `<div class="col">` |
| 右に画像・左にテキスト | `<!-- _class: text-image -->` + `text-main` / `image-support` |
| 上に画像・下にメッセージ | `<!-- _class: image-message -->` + `image-wide` / `message` |
| 強調色（青） | `<span class="text-blue">` |
| 強調色（赤） | `<span class="text-red">` |
| ボックス装飾 | `marp-box marp-box-blue/pink/yellow` |
| 脚注・出典 | `<p class="footnote">...</p>` |

画像素材は `slides/YYYYMMDD_title/materials/` に置いて、`materials/ファイル名` で参照する。

### 4-3. コンパイル

```bash
task compile -- slides/YYYYMMDD_title
```

`main.html` と `main.pdf` が同ディレクトリに出力される。

### 4-4. 完了報告

- PDFのパスをユーザーに伝える
- スライドを作る中で気になった点（情報不足、構成上の迷い、改善できそな点）があれば一言添える
